Dashboard principal Admin Panel Configuration

Page principale — Dashboard de surveillance

La page principale affiche en temps reel l'etat du serveur et des services. Les services DOWN sont masques (un compteur discret en bas).

Informations affichees

  • Modele charge : le nom du modele LLM actif (detecte via /v1/models)
  • CPU Load / RAM : utilisation systeme
  • Services Status : statut de chaque service (vert = UP, rouge = DOWN, jaune = SLOW, gris = LOADING)
  • Token rates : prompt et generation tokens par seconde (via /metrics)
  • GPU Cards : temperature, VRAM, consommation electrique, SM/mem clocks, throttling
  • Logs : derniers logs de chaque service, onglets separes

Page Admin — Controle des services

Accessible via /admin/panel, protegee par mot de passe. Permet de demarrer, arreter et forcer l'arret des services.

LLM Engine

Dropdown listant tous les services LLM avec un service systemd. Selectionnez un modele puis cliquez Demarrer. Le service concurrent dans le meme groupe exclusif est arrete automatiquement.

Projets et auxiliaires

Cartes individuelles pour les services hors LLM Engine (Ollama, proxies, STT...). Chaque carte affiche le role (LLM, Projet, Auxiliaire) et le statut health (UP/DOWN).

Boutons

  • Demarrer : sudo systemctl start <unit>
  • Arreter : sudo systemctl stop <unit>
  • Redemarrer : stop puis start
  • Force Kill : fuser -k <port>/tcp + kill GPU

Page Configuration — /admin/config

Interface web pour gerer les services sans editer config.yaml a la main.

Onglet Audit

Scanne la machine pour detecter : backends installes (vLLM, llama.cpp, ik_llama.cpp, SGLang, Ollama, LM Studio), modeles (~/models/ et ~/.cache/lm-studio/models/), ports ouverts, services systemd, conda envs. Cliquez sur un element pour pre-remplir le formulaire.

Onglet Ajouter

Formulaire complet pour definir un service : cle, role (LLM Engine / Projet / Auxiliaire), backend, nom, port, URL, systemd unit, modele, log, groupe exclusif. La commande de lancement (ExecStart) est editable librement — un template par defaut est fourni par backend, modifiez-le selon vos besoins. Boutons : apercu YAML, generer .service, installer le service, sauvegarder.

Onglet Services

Tableau des services definis. Edition et suppression. Lors de l'edition, le vrai ExecStart du fichier .service est charge (pas le template).

Changer le mot de passe admin

  • Executez python change_admin_password.py depuis le repertoire d'installation
  • Le script detecte automatiquement l'environnement conda
  • Vous pouvez aussi definir admin.password_hash dans config.yaml

Services systemd

Chaque LLM Engine est gere par un service systemd. Commandes utiles :

  • sudo systemctl start vllm-qwen27b.service — demarrer
  • sudo systemctl stop vllm-qwen27b.service — arreter
  • journalctl -u vllm-qwen27b.service -f — logs en direct
