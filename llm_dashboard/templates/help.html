Page principale — Dashboard de surveillance

La page principale affiche en temps reel l'etat du serveur et des services LLM. Elle se recharge automatiquement toutes les quelques secondes.

Informations affichees

  • Modele charge : le nom du modele LLM actif sur le port 8080 (si detecte)
  • CPU Load : charge CPU du serveur en pourcentage
  • RAM Usage : memoire RAM utilisee / totale
  • Services Status : statut de chaque service (vert = actif, rouge = arrete, jaune = lent, gris = chargement)
  • GPU Cards : pour chaque GPU : temperature, VRAM utilisee, consommation electrique
  • Logs : derniers logs de chaque service, avec onglets par service

Couleurs des statuts

StatutCouleurSignification
UPVertService actif et repond correctement
DOWNRougeService arrete ou ne repond pas
SLOWJauneService actif mais temps de reponse eleve
LOADINGGrisService en cours de chargement (vLLM : 60-120s)

Jauges GPU

Les barres de progression VRAM et puissance changent de couleur selon les seuils configures :

  • Vert : en dessous du seuil d'avertissement
  • Jaune : entre seuil d'avertissement et seuil critique
  • Rouge : au-dela du seuil critique

Page Admin — Controle des services

La page Admin permet de demarrer, arreter et redemarrer les services LLM et auxiliaires. Elle est protegee par mot de passe. Pour y acceder, cliquez sur Admin Panel depuis la page principale.

Regle principale

Un seul service LLM peut tourner a la fois sur le port 8080. Demarrer un LLM arrete automatiquement le LLM precedemment actif.

Les services auxiliaires et Ollama sont independants : ils ne partagent pas le port 8080 et peuvent tourner en meme temps qu'un LLM sur 8080.

Boutons de controle

  • Demarrer : lance le service (attend que le port soit ouvert)
  • Arreter : arrete proprement le service (systemctl stop, puis fuser -k si necessaire)
  • Redemarrer : arrete puis relance le meme service
  • Force Kill : tue les processus GPU restants (SIGKILL) — a utiliser si Arreter ne suffit pas
  • Arreter tous les LLM : arrete tous les LLM sur le port 8080

Temps de chargement

Les modeles LLM mettent du temps a se charger en memoire GPU :

  • ik_llama.cpp / llama.cpp : 30 a 60 secondes
  • vLLM : 60 a 120 secondes (le modele FP8 + compilation CUDA)

Pendant le chargement, le statut affiche LOADING. Il faut attendre que le port 8080 soit ouvert pour utiliser l'API.

VRAM

La section VRAM affiche l'occupation memoire des GPUs. Apres un arret, la VRAM peut mettre quelques secondes a se liberer. Si des processus GPU restent bloques, utilisez Force Kill.

Changer le mot de passe admin

Pour modifier le mot de passe d'acces au panel Admin :

  • Executez python3 /opt/dashboard-llm/change_admin_password.py
  • Suivez les instructions pour saisir le nouveau mot de passe
  • Le hash est mis a jour dans /opt/dashboard-llm/config.yaml

API Endpoints

URLs utiles pour verifier les services :

  • http://<serveur>:5000/ — Dashboard principal
  • http://<serveur>:5000/admin — Page de connexion Admin
  • http://<serveur>:5000/help — Cette page
  • http://<serveur>:5000/api/data — Donnees JSON (statut systeme)
  • http://<serveur>:5000/api/admin/status — Statut admin (auth requise)
  • http://<serveur>:5000/health — Healthcheck
  • http://<serveur>:8080/v1/models — Modeles LLM disponibles
  • http://<serveur>:8080/v1/chat/completions — API chat LLM
